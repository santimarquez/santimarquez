# Cursor Quick Start

Quick reference for using Cursor with this personal leadership site.

## Setup

- **CURSOR.md** – Project context (this repo)
- **.cursorrules** – Guidelines (Astro, Tailwind, Motion One, i18n, design system)
- **ai/** – Context, docs, skills, agents, prompts

## Start Using Cursor

### Example prompts

```
Add a new section to the hero with one more metric
Implement the experience timeline with collapsible roles
Add Spanish translations for the Technical Stack section
Create the Contact form with success state animation
Optimize meta tags and add Person schema for SEO
Add dark mode toggle with smooth transition
Create a simple SVG architecture diagram for the AlertaCoches project
```

### By area

**Layout & design**
- “Use the design system: minimal, neutral palette, one accent”
- “Add a section reveal animation with Motion One”
- “Ensure this section is WCAG AA compliant”

**i18n**
- “Add ES translation for [section]”
- “Wire the language switcher to i18n”

**Content & copy**
- “Rewrite this as an impact statement and add a metric”
- “Make this bullet recruiter-scannable”

**DevOps**
- “Check Dockerfile and docker-compose for Traefik”
- “Add missing env vars to the production build”

## Key paths

- Components: `src/components/`
- Pages: `src/pages/`
- Layouts: `src/layouts/`
- i18n: `src/i18n/`
- Styles: `src/styles/`
- AI context: `ai/context/`

## Docs

- **Full context:** `CURSOR.md`
- **Rules:** `.cursorrules`
- **AI guide:** `ai/README.md`
- **Cursor setup:** `ai/docs/CURSOR_SETUP.md` (if present)

Open Cursor and start from `CURSOR.md` and `.cursorrules` when working on this repo.
